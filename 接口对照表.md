# 云台功能接口对照表

本文档整理当前项目中各项云台功能对应的：

- CLI 命令行接口
- C 语言函数接口
- 串口原始指令
- 功能说明

协议采用 7 字节 Pelco-D 帧格式：

```text
FF 地址 命令1 命令2 数据1 数据2 校验
```

校验和算法：

```text
checksum = address + command1 + command2 + data1 + data2
```

默认地址为 `0x01`。

## 设备参数

根据你提供的设备参数，当前云台能力范围如下：

| 参数 | 数值 |
|---|---|
| 水平旋转角度 | `0° ~ 350°` |
| 垂直旋转角度 | `-32° ~ +23°` |
| 水平旋转速度 | `14.3°/s` |
| 垂直旋转速度 | `1.7°/s` |

这些参数说明：

- `pan` 表示水平角度，对应范围应落在 `0° ~ 350°`
- `tilt` 表示俯仰角度，对应范围应落在 `-32° ~ +23°`
- 当前代码里 `tilt=1509.78` 这类结果明显超出设备物理范围，说明现有俯仰角解析公式与实际设备协议不一致
- 速度统一使用角速度单位 `°/s` 表达
- 设备参数中的速度表示机械角速度能力上限
- 协议命令中的 `speed` 参数当前仍是控制档位，尚未建立与 `°/s` 的准确换算关系

## 二次集成说明

- `main.c` 现在只作为独立调试用的 CLI 入口，不再是功能承载主体。
- 对外给其他同事集成时，直接包含头文件 `include/gimbal.h`，链接库 `gimbal` 即可，不需要调用 `main.c`。
- 推荐按下面的生命周期调用：
  1. `gimbal_config_t config;`
  2. `gimbal_config_init_default(&config);`
  3. 按需覆盖 `config.device / config.baudrate / config.address`
  4. `gimbal_init(&gimbal, &config);`
  5. 调用各业务接口
  6. `gimbal_deinit(&gimbal);`

### 推荐对外接口

| 接口 | 说明 |
|---|---|
| `gimbal_config_init_default()` | 初始化默认配置，默认值为 `/dev/ttyS8`、`2400`、`0x01` |
| `gimbal_init()` | 按配置打开串口并初始化云台句柄 |
| `gimbal_deinit()` | 关闭串口并释放云台句柄 |
| `gimbal_move_deg_per_sec()` | 按角速度 `°/s` 调用转动接口，适合作为上层统一调用入口 |
| `gimbal_move()` | 底层速度码接口，直接使用协议速度码 |
| `gimbal_move_to_position()` | 移动到指定水平/俯仰角，并轮询直到到位或超时 |
| `gimbal_initialize_presets()` | 按预置位数组批量完成“移动到位 + 保存预置位” |
| `gimbal_scan_set_left_limit()` | 完成“移动到左起点 + 到位校验 + 保存左限位” |
| `gimbal_scan_set_right_limit()` | 完成“移动到右起点 + 到位校验 + 保存右限位” |

## 角度换算说明

### 水平角度

当前实现中：

```text
units = degrees / 0.8
degrees = units * 0.8
```

例如：

- `180° -> 225 -> 0x00E1`

对应设置水平角度指令为：

```text
FF 01 00 4B 00 E1 2D
```

结合设备参数，水平角度更合理的有效范围应视为：

```text
0° ~ 350°
```

### 俯仰角度

当前实现中：

```text
units = degrees * 100
degrees = units / 100
```

例如：

- `90.00° -> 9000 -> 0x2328`

对应设置俯仰角度指令为：

```text
FF 01 00 4D 23 28 99
```

其中：

- `0x23 0x28` 表示 `90.00 * 100`
- `0x99` 为校验和

结合设备参数，俯仰角更合理的有效范围应视为：

```text
-32° ~ +23°
```

这意味着：

- 当前代码中的俯仰角换算公式只是基于早期样例实现
- 它并不能保证适配你现在这台设备
- 如果查询结果超出 `-32° ~ +23°`，应优先怀疑协议解析公式，而不是设备真的转到了这个角度

## 速度说明

你提供的设备机械速度参数为：

- 水平：`14.3°/s`
- 垂直：`1.7°/s`

文档中的速度单位统一为角速度 `°/s`。

需要区分两类“速度含义”：

1. 设备机械角速度

- 这是云台实际转动能力
- 表示在正常工作条件下的角速度上限，单位为 `°/s`

2. 协议速度控制量

- 例如左转指令 `FF 01 00 04 XX 00 CS` 里的 `XX`
- 当前 CLI 接收的 `--speed` 参数单位是 `°/s`
- 程序会依据设备最大角速度把 `°/s` 换算成协议控制量 `XX`
- 这个换算是当前项目的工程映射，并不代表设备协议文档明确定义了线性关系

因此当前代码中的 `--speed`：

- 可以控制目标角速度的快慢
- 输入单位是 `°/s`
- 水平方向建议范围为 `0 ~ 14.3`
- 垂直方向建议范围为 `0 ~ 1.7`
- 程序会把输入角速度裁剪到设备最大能力范围内

## 预置位初始化

当前程序提供显式接口来初始化预置位，不再在第一次启动时自动执行。

### 配置文件

预置位坐标表文件：

- [preset_positions.yaml](D:\traework\云台\code\preset_positions.yaml)

支持的预置位编号：

- `1 ~ 16`
- `33 ~ 48`

每个预置位需要配置：

- `pan`：水平角度
- `tilt`：垂直角度

### 调用方式

执行以下命令后，程序会开始按配置表设置预置位：

```bash
./build/gimbal_cli --device /dev/ttyS8 init-presets

initialize_presets
```

### 执行流程

程序执行 `init-presets` 时会：

1. 读取 `preset_positions.yaml`
2. 按顺序处理 `1~16`、`33~48`
3. 先移动到配置的水平角度和垂直角度
4. 查询当前云台角度
5. 若水平和垂直角度都满足误差不超过 `0.1°`，则执行对应预置位设置
6. 全部完成后结束命令执行

### 注意事项

- 如果 `preset_positions.yaml` 缺少任一预置位配置，程序会直接报错退出
- 如果云台在超时时间内未到达目标坐标，程序会报错退出
- 当前到位判定误差为 `0.1°`
- 只有显式调用 `init-presets` 时才会执行预置位初始化

## 线扫速度接口更新

- `scan-speed <5-13>` 现在表示“设置线扫速度”。
- 实际发送顺序分两步：
  1. 先发 `22+设置`：`FF 01 00 03 00 16 1A`
  2. 再发一条速度指令：`FF 01 00 03 00 PP CS`
- 速度档位共 `9` 档，对应 `5+设置` 到 `13+设置`，即 `PP=0x05~0x0D`。
- 按档位递增理解速度递增：
  - 最慢是 `5+设置`：`FF 01 00 03 00 05 09`
  - 最快是 `13+设置`：`FF 01 00 03 00 0D 11`
